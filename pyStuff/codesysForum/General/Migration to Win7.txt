Yes, i know, i'm in contact with the support staff. It seems my plc has problem to be programmed with newer versions, even if the compiler version is setted to my used version.

One thing you can do is save off your libraries of the old version of CoDeSys in to an archive file or just another folder on the computer. Install the new version of CoDeSys but only reference the old libraries in your new project. This typically works. I have done this with some Schneider software that was using as old a version as you and it works fine. The issue is 3S makes changes but doesn't rename the library files. So when you install or even reference the newest libraries it may not compile. Every time I upgrade CoDeSys I save off the version of 3S libraries into another folder like V2372, V2396..etc. Then I can try the new version and see if I have any problems with 3s latest release. If there is I just reference back to the older libraries. This has been working for me for about 5 years now.
