Hi,i had the same Problem.attched you'll find the answer from 3S hotline, it solve my problem !offensichtlich kommt es zu einem Versionskonflikt von Codemeter. Bei uns in der Datenbank habe ich folgende Abhilfe gefunden: Diagnose:Im Verzeichnis C:\Windows\System32 bzw. C:\Windows\SysWOW64 (bei 64 Bit Systemen) befindet sich eine Datei WibuCm32.dll. Diese Datei sollte die gleiche Version wie Wibu Codemeter haben.Im Fehlerfall liegt hier eine Datei mit der Prdouktversion 5.10, auf dem Rechner ist jedoch Codemeter 6.20 installiert. Fehlerbehebung:·         Version von Wibu Codemeter notieren·         Wibu Codemeter deinstallieren·         Die gleiche Version von Wibu Codemeter wieder installieren.Beim ungepackten CODESYS V3.5 SP9 Setup liegt das Codemeter Setup in den Unterverzeichnissen  ISSetupPrerequisites\{2CC15EAC-F048-465D-A093-BCE8BE8394B3} (für 32 Bit)bzw.  ISSetupPrerequisites\{7467EC5D-47F5-4CA7-B27C-04CF3197EB9B} (für 64 Bit)·         Kontrolle der Dateiversion von WibuCm32.dll
